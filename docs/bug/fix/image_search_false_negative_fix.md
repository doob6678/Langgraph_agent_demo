# 图片检索误报“未找到”修复方案

## 1. 问题背景

当前线上现象是：  
- 文搜图、图搜图在接口返回 `images` 字段里已经有命中结果；  
- 但 `response` 文本仍出现“未找到相关图片”。  

这会让用户误判为检索失败，属于结果一致性问题。

## 2. 已确认根因

本问题不是 Milvus 无数据，也不是 CLIP 不可用，核心是“结果双链路不一致”：

1. 检索链路能拿到真实命中并写入结构化结果（`state.images`）。  
2. 大模型回答链路依赖工具文本结果。  
3. 当工具文本结果未正确使用同一批命中数据时，会返回“未找到相关图片”。  
4. 最终出现：结构化结果有图，但自然语言回答说没有图。

## 3. 改造目标

1. 只要图片检索有命中，工具文本与最终回答都必须体现“已命中”。  
2. `response` 与 `images` 使用同一数据源，禁止分叉。  
3. 对低分命中统一阈值策略，避免“弱相关结果被当成未命中”的歧义。

## 4. 改哪些文件

1. `backend/agent/tool_ex/rag_image_search_strategy.py`  
   - 统一组装可复用的图片命中列表（包含 id、image_uri/filename、score、content）。  
   - 显式把同一批命中结果传给工具层与状态层，确保来源一致。

2. `backend/agent/tool_ex/tools.py`  
   - 调整 `rag_image_search` 文本生成逻辑：优先根据传入命中列表输出，不再在有结果时返回“未找到相关图片”。  
   - 输出中包含 `filename/image_uri + score + content摘要`，用于给大模型稳定生成回答。  
   - 为“无命中”与“低于阈值命中”分别给出明确文案。

3. `backend/agent/tool_ex/analyze_image_strategy.py`（图搜图链路）  
   - 与文搜图保持同样的数据拼装规范，确保图搜图输出风格一致。  
   - 将相似图命中列表写入状态并同步给工具文本。

4. `backend/agent/tool_ex/state_tool_runner.py`  
   - 校验工具结果回写顺序，保证工具文本不会覆盖已存在的有效命中状态。  
   - 补充统一结果规范化处理，避免字段缺失导致回答误判。

## 5. 怎么改

### 5.1 统一结果模型

新增统一命中结构（内部约定）：
- `id`
- `image_uri`
- `filename`
- `score`
- `content`

所有策略层先把原始结果转换为该结构，再同时用于：
- `state.images`（前端展示）
- 工具文本拼接（LLM回答依据）

### 5.2 统一命中判定

引入统一判定逻辑：
- `results 为空` -> 无命中  
- `results 非空且全部低于阈值` -> 低相关命中（提示“找到低相关结果”）  
- `存在高于阈值结果` -> 命中成功（输出前 N 条）

### 5.3 文本拼接策略

工具文本输出格式固定为：
- 命中条数  
- 每条的 `filename/image_uri + score + content摘要`

这样大模型回答将基于可解释的同源数据，避免误报。

## 6. 测试计划

### 6.1 启动与健康检查

1. 重启后端服务。  
2. 访问健康接口确认服务可用。  
3. 确认 Milvus 连接正常、图片集合可查询。

### 6.2 冒烟用例

1. 文搜图命中用例  
   - 输入已知关键词（如历史入库图片描述词）。  
   - 预期：`images` 非空，`response` 明确“已找到相关图片”。  

2. 图搜图命中用例  
   - 上传已入库图片做相似检索。  
   - 预期：Top1 命中自身或同类图，`response` 不出现“未找到”。  

3. 无命中用例  
   - 输入明显无关关键词。  
   - 预期：`images` 为空且 `response` 明确无命中。  

4. ACL 隔离用例  
   - 以不同 `user_id/dept_id` 检索同关键词。  
   - 预期：仅返回有权限图片，不泄漏他人私有图。

### 6.3 一致性断言

对每个用例增加一致性校验：
- 若 `images` 非空，则 `response` 不得包含“未找到相关图片”。  
- 若 `response` 声称命中，则 `images` 必须有对应条目。

## 7. 验收标准

1. 文搜图、图搜图主流程稳定命中，不再出现“有图却说没找到”。  
2. `response` 与 `images` 全部用例一致。  
3. ACL 行为符合预期，无跨用户越权返回。  
4. 冒烟通过后再进入回归验证。
